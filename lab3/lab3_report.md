# Отчёт по лабораторной работе 3

## 1. Описание деплоя

Цель — развернуть Telegram-бота трекера привычек в облаке так, чтобы им можно было пользоваться без локального запуска `python bot.py` на своём компьютере. Выбрана платформа **[Railway](https://railway.app/)**: деплой из репозитория GitHub, переменные окружения в панели, процесс запуска через **`Procfile`** (`worker: python bot.py`).

Режим работы с Telegram API — **long polling** (`run_polling`), отдельный публичный URL и webhook для этой лабораторной не требуются.

## 3. Подготовка проекта

Сделано в репозитории:

- **`requirements.txt`** — зафиксированы `python-telegram-bot`, `python-dotenv`, `matplotlib`.
- **`.gitignore`** — исключены `.env`, `*.db`, `__pycache__`, прочий служебный мусор.
- **`.env.example`** — пример с `BOT_TOKEN=` без секретов.
- **`Procfile`** — строка `worker: python bot.py` для Railway.
- **`runtime.txt`** — версия Python для согласованной сборки.
- **`README.md`** — локальный запуск и пошаговый деплой на Railway; замечание про **SQLite** (данные в контейнере при пересборке без volume могут теряться).
- В **`bot.py`**: токен только из `os.environ` / `.env` через `load_dotenv()`; логирование **INFO**; временные файлы графиков — в системном temp; сообщение об отсутствии `BOT_TOKEN` с подсказкой про Railway Variables.
