# Habit tracker — Telegram-бот

Репозиторий курса [Vibe Coding: AI-боты для бизнеса](https://github.com/itmo-ict-faculty/vibe-coding-for-business).

Бот помогает вести привычки: добавление, список, отметка за сегодня, удаление, статистика, **очки и уровни**, **серии**, экран **«Мой прогресс»** и **столбчатый график** за 7 дней. Управление — **кнопками**; текстом вводится только название новой привычки после «Создать привычку».

**Хранение:** SQLite — файл `database.db` (создаётся при запуске, в git не коммитится). При первом запуске, если рядом есть старый `data.json` из лабы 1 и таблица привычек пустая, данные переносятся в БД. При обновлении со старой схемы без полей геймификации выполняется **миграция**: добавляются колонки `total_points` и `best_streak`, очки и рекорды серий пересчитываются по истории.

**Секреты:** токен только в переменной окружения `BOT_TOKEN` (локально через `.env`, на хостинге через панель). В коде и в git токена нет.

## Требования

- Python 3.10+ (для продакшена см. `runtime.txt`)
- Токен бота от [@BotFather](https://t.me/BotFather)
- Зависимости: `python-telegram-bot`, `python-dotenv`, `matplotlib` — см. `requirements.txt`

## Локальная установка и запуск

```bash
cd 2025-2026-chatbots-U4125-SEMENOV_A_A
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# В .env укажите: BOT_TOKEN=...
python bot.py
```

Точка входа: **`python bot.py`** (или процесс из `Procfile` на Railway). Логирование: **INFO** в консоль (запуск, путь к БД, ошибки).

## Деплой на [Railway](https://railway.app/)

Бот использует **long polling** (`run_polling`) — открытый HTTP-порт и webhook не обязательны.

1. Залейте репозиторий на GitHub и подключите его в Railway (**New Project** → **Deploy from GitHub**).
2. Укажите переменную окружения **`BOT_TOKEN`** в разделе **Variables** (значение — токен от BotFather). Файл `.env` в облако не коммитится и не обязателен.
3. Команда старта: в корне есть **`Procfile`** с строкой `worker: python bot.py` — Railway подхватит процесс `worker`.
4. Дождитесь успешного деплоя и проверьте бота в Telegram.

**SQLite на Railway:** файл `database.db` создаётся в файловой системе контейнера. При **пересборке/новом деплое** без постоянного тома данные могут обнулиться. Для сохранения БД между деплоями добавьте **Volume** и при необходимости задайте путь к файлу через переменную (в текущей версии путь фиксирован рядом с `bot.py`; для продакшена имеет смысл вынести путь в переменную окружения — опционально).

**Графики:** временные PNG создаются в системном каталоге временных файлов и удаляются после отправки в Telegram.

## Геймификация и логика

| Элемент | Правило |
|---------|---------|
| Очки | +10 за каждую **новую** отметку за день (повтор в тот же день не даёт очков) |
| Уровень | 1 при 0–99 очков, далее +1 уровень каждые 100 очков |
| Текущая серия | Подряд идущие дни с отметкой, цепочка заканчивается сегодня или вчера; пропуск дня обнуляет текущую серию |
| Лучшая серия по привычке | Максимум из текущей серии после отметок и истории; хранится в БД |

## Интерфейс

| Действие | Как |
|----------|-----|
| Старт | `/start` |
| Создать привычку | «Создать привычку» → текст названия |
| Список | «Список» |
| Отметить сегодня | «Отметить» → inline-кнопка (очки и серии в ответе) |
| Статистика | «Статистика» — по привычкам + лидеры за 7 дней |
| Мой прогресс | «Мой прогресс» — привычки, всего выполнений, очки, уровень, лучшая серия |
| График | «График» — столбчатый график отметок за 7 дней (изображение) |
| Удалить | «Удалить» → выбор привычки |

## Безопасность

Не коммитьте `.env` и не публикуйте токен. При утечке — отзовите токен в BotFather и создайте новый.
